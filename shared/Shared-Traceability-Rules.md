# Shared Traceability Rules

## Universal traceability chain

```text
Stage 4 final artifact
→ Stage 3 fabricated artifact
→ Stage 2 work package
→ Stage 1 design entity
→ Stage 1 decision / requirement / asset / acceptance criterion
```

## Stage 1 matrix

| Feature ID | Feature Name | Requirement IDs | Acceptance Criteria IDs | Screen / Interface IDs | Asset IDs | Data / API IDs | Related Decisions | Notes |
|---|---|---|---|---|---|---|---|---|

## Stage 2 matrix

| Stage 1 Entity ID | Stage 1 Entity Type | Stage 1 Entity Name | Stage 2 Work Package IDs | Stage 3 Prompt IDs | File / Artifact IDs | QA Package IDs | Traceability Status |
|---|---|---|---|---|---|---|---|

## Stage 3 matrix

| Stage 1 Entity ID | Stage 2 Work Package ID | Stage 3 Fabrication Result ID | Artifact IDs | Validation IDs | Status | Notes |
|---|---|---|---|---|---|---|

## Stage 4 matrix

| Trace ID | Stage 1 Source IDs | Stage 2 Work Package IDs | Stage 3 Artifact IDs | Stage 4 Final Artifact IDs | Validation IDs | Final Status | Notes |
|---|---|---|---|---|---|---|---|

## Gap handling

When a traceability gap is found, identify the affected item, identify missing source authority, classify severity, repair if authority is clear, or mark blocked, review-gated, placeholder-complete, external-validation-required, excluded, or not applicable.
