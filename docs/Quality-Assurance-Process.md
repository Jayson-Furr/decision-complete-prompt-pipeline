# Quality Assurance Process

Before publishing a release, run:

```bash
python tools/run-all-checks.py
python tools/build-manifest.py
python tools/build-release-manifest.py
```

The release should not be published unless repository structure, prompt files, boundary markers, configuration examples, controlled vocabularies, schemas, examples, decision ledger rules, machine-readable companion rules, and Markdown links pass validation.

Manual review should also verify that the repository does not claim deployment, app store approval, legal approval, rights clearance, certification, security certification, or compliance status.

## QA scope boundary

Repository QA validates the package itself. Product QA for artifacts generated by the prompts remains a downstream Stage 4 responsibility and may require external tools, target runtimes, professional review, or user authorization.
