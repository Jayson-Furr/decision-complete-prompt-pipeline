
# Stage 2 Overview — Execution Planning

Stage 2 is the execution-planning authority stage of DecisionComplete Prompt Pipeline.

Stage 2 transforms the locked Stage 1 design artifact into a **100% Execution-Ready Fabrication Package Suite**.

## Stage 2 output

```text
100% Execution-Ready Fabrication Package Suite
```

## Stage 2 controls

```text
Production decomposition, work package boundaries, work package registry, generated Stage 3 prompts, contractor briefs, dependency graph, execution order, expected artifact manifest, QA validation matrix, placeholder and review-gated handling, blocked item handling, and Stage 3 handoff rules.
```

## Stage 2 does not control

```text
Product redesign, feature changes, visual design changes, asset design changes, final code fabrication, final binary asset creation, final validation, deployment, publishing, or release.
```

## Stage 2 prompt files

```text
prompts/Stage-2-Prompt-1-No-Questions-Full-AI-Execution-Planning.md
prompts/Stage-2-Prompt-2-Low-Stakes-Execution-Interview.md
prompts/Stage-2-Prompt-3-Medium-Stakes-Execution-Interview.md
prompts/Stage-2-Prompt-4-High-Stakes-Execution-Interview.md
```

## Recommended default

```text
Stage-2-Prompt-3-Medium-Stakes-Execution-Interview.md
```

Use Medium-Stakes mode for most projects because work package boundaries, dependencies, artifact manifests, and QA depth are usually material execution decisions.
