
# Stage 1 Overview — Design Specification

Stage 1 is the design authority stage of DecisionComplete Prompt Pipeline.

Stage 1 transforms rough user context, attached files, images, configuration, and preferences into a **100% Decision-Complete Digital Product Design Specification Suite**.

## Stage 1 output

```text
100% Decision-Complete Digital Product Design Specification Suite
```

## Stage 1 controls

```text
Product type, product scope, features, screens, controls, workflows, user stories, visual design, asset design, audio design, animation design, data model, API / interface design, technical architecture, requirements, acceptance criteria, placeholders, review gates, source provenance, and Stage 2 handoff rules.
```

## Stage 1 does not control

```text
Work package creation, Stage 3 prompt generation, contractor assignment, code fabrication, binary asset creation, final validation, packaging, deployment, publishing, or release.
```

## Recommended default

```text
Stage-1-Prompt-3-Medium-Stakes-Design-Interview.md
```

Use Medium-Stakes mode for most public-repository and real product work because it balances speed with control and auditability.
