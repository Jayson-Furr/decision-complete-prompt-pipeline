# Governance

DecisionComplete Prompt Pipeline uses a lightweight maintainer-led governance model.

## Project identity

```text
Project name: DecisionComplete Prompt Pipeline
Repository name: decision-complete-prompt-pipeline
Version: 1.0.0
```

## Non-negotiable invariants

```text
Stage 1 controls design.
Stage 2 controls execution planning.
Stage 3 controls fabricated outputs.
Stage 4 controls validation, repair classification, packaging, and final handoff.
```

Maintainers should preserve human-readable-first artifacts, stable IDs, artifact boundary markers, traceability, decision ledgers, placeholders, review gates, high-stakes handling, and the no-unauthorized-side-effect rules.
