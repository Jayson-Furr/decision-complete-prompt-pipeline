# Support

For a public GitHub repository, use GitHub Issues for questions, documentation problems, defects, or enhancement requests.

## Before Opening an Issue

Please check:

- `README.md`
- `docs/Quick-Start.md`
- `docs/FAQ.md`
- `docs/Troubleshooting.md`
- `docs/Validation-Tools.md`

## Useful Issue Details

Include:

- The stage and prompt file involved.
- The prompt mode.
- Configuration files used.
- Whether the issue involves placeholders, review gates, external-tool-required items, or external-validation-required items.
- A minimal reproduction if possible.

Do not include secrets, credentials, private keys, private customer data, or proprietary project data in public issues.
