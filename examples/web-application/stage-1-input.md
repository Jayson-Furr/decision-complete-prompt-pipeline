# Stage 1 Input Packet — Web Application Example

## Product name

```text
CivicIssue Board
```

## Product type

```text
Web application
```

## Short description

A web application for small civic teams to collect, triage, assign, and track public infrastructure issues reported by residents.

## Preferred stack

```text
Frontend: React + TypeScript
Backend: Node.js service API
Database: PostgreSQL
Styling: CSS Modules or design tokens
Deployment: not authorized in this example
```

## Core screens

```text
- public issue submission form;
- issue confirmation page;
- public issue list;
- staff login;
- staff issue queue;
- issue detail page;
- assignment panel;
- admin category management;
- settings page;
- report export page.
```

## High-stakes review requirements

```text
- Authentication and authorization must be reviewed.
- Personal data handling must be reviewed.
- Location data handling must be reviewed.
- Public release and production deployment are not authorized.
- Do not claim government approval or official municipal status.
```
