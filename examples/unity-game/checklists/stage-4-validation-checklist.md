# Stage 4 Validation Checklist — Starfall Cartographer

Use this checklist after generating the Stage 4 final handoff package.

| Check | Expected Handling |
|---|---|
| Stage 3 artifact inventoried | Stage 4 intake summary includes Stage 3 source inventory. |
| Stage 1 design lock preserved | No Stage 4 repair changes the product design. |
| Stage 2 execution lock preserved | No Stage 4 repair changes work package intent. |
| Stage 3 fabrication record preserved | All fabricated, partial, blocked, review-gated, and external-tool items are classified. |
| Final artifact manifest complete | Every Stage 3 artifact is included, excluded, blocked, review-gated, external-tool-required, external-validation-required, or not applicable. |
| Decision ledger complete | Active logging threshold is satisfied. |
| Traceability complete | Final artifacts trace to Stage 3, Stage 2, and Stage 1 where applicable. |
| External validation honest | No unrun build/test/import/deployment is claimed as passed. |
| High-stakes issues preserved | Security, privacy, rights, release, credential, and deployment issues are review-gated or blocked as needed. |
| Final handoff status accurate | Ready, Ready with Warnings, Review-Gated, Blocked, or External Validation Required is justified. |

- Unity Editor import validation
- C# compile validation
- UI Toolkit import validation
- Play Mode smoke test
